## Core Security & Technical Resources (Authoritative Links)

### **Secure by Design & Security Architecture**

- **OWASP Secure-by-Design Framework**  
  https://owasp.org/www-project-secure-by-design-framework/  
  *A comprehensive framework for building software with security principles embedded from the start.*

- **OWASP Product Security Guide**  
  https://owasp.org/www-project-product-security-guide/  
  *Guidance covering secure product development, threat modelling, lifecycle management, and long-term maintenance.*

- **OWASP Mobile Application Security Design Guide (MASDG)**  
  https://owasp.org/www-project-mobile-application-security-design-guide/  
  *A set of best practices for creating secure mobile applications from design through implementation.*

- **OWASP Security Principles (Foundations)**  
  https://devguide.owasp.org/fa/02-foundations/03-security-principles/  
  *A concise reference of core security design principles such as least privilege, complete mediation, and fail-safe defaults.*

- **Secure-by-Design (OWASP PDF)**  
  https://wiki.owasp.org/images/e/e9/Sulatycki_Secure-By-Design_140319.pdf  
  *An in-depth PDF exploring secure design patterns, misuse cases, and practical approaches to threat modelling.*

  https://www.raspberrypi.com/documentation/

- **Raspberry Pi Security & Vulnerability Disclosure**
